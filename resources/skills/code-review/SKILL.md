# Code Review Skill

## 목적
코드 변경사항을 체계적으로 리뷰하여 품질, 보안, 성능 문제를 식별합니다.

## 리뷰 절차

1. **변경 범위 파악**: 수정된 파일 목록과 diff를 확인합니다.
2. **로직 검증**: 비즈니스 로직이 의도대로 동작하는지 확인합니다.
3. **에러 처리**: 예외 상황과 엣지 케이스가 처리되는지 확인합니다.
4. **보안 검토**: OWASP Top 10 취약점 여부를 점검합니다.
5. **성능 확인**: N+1 쿼리, 불필요한 반복, 메모리 누수 가능성을 점검합니다.

## 체크리스트

- [ ] 함수/메서드 이름이 동작을 명확히 표현하는가
- [ ] 입력 검증이 시스템 경계(사용자 입력, 외부 API)에서 수행되는가
- [ ] 에러 메시지가 디버깅에 유용한 정보를 포함하는가
- [ ] 테스트가 핵심 경로를 커버하는가
- [ ] 하드코딩된 값이 설정으로 분리되었는가

## 출력 형식

리뷰 결과는 다음 형식으로 작성합니다:

```
## 리뷰 요약
- 심각도: Critical / Warning / Info
- 파일: {파일 경로}:{라인}
- 설명: {문제 설명}
- 제안: {수정 방안}
```
