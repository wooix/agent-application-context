# Documentation Writing Skill

## 목적
코드와 시스템에 대한 명확하고 유용한 기술 문서를 작성합니다.

## 문서 구조

1. **개요**: 무엇을 하는지 한 문장으로 설명
2. **사용법**: 가장 일반적인 사용 예시
3. **API 레퍼런스**: 파라미터, 반환값, 예외
4. **예제**: 실행 가능한 코드 예시

## 작성 원칙

- 독자의 컨텍스트를 가정하지 않습니다
- 코드 예제는 실제 실행 가능해야 합니다
- "왜(why)"를 "무엇(what)"보다 우선 설명합니다
- 한국어와 영어 용어를 적절히 혼용합니다 (코드 식별자는 영어)
