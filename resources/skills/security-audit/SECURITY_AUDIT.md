# Security Audit Skill

## 목적
코드베이스의 보안 취약점을 체계적으로 식별하고 보고합니다.

## 감사 영역

### 1. 입력 검증
- SQL 인젝션, XSS, 명령어 인젝션 가능성 점검
- 사용자 입력이 적절히 이스케이프/검증되는지 확인

### 2. 인증/인가
- 인증 우회 가능성 점검
- 권한 검사가 모든 엔드포인트에 적용되는지 확인

### 3. 데이터 보호
- 민감정보(비밀번호, 토큰, API 키)가 평문으로 저장/전송되지 않는지 확인
- 로그에 민감정보가 노출되지 않는지 확인

### 4. 의존성 보안
- 알려진 취약점이 있는 패키지 사용 여부 점검

## 심각도 분류

| 등급 | 설명 |
|------|------|
| Critical | 즉시 악용 가능한 취약점 |
| High | 조건부 악용 가능, 빠른 수정 필요 |
| Medium | 잠재적 위험, 계획적 수정 |
| Low | 모범 사례 미준수, 개선 권장 |
